# kairo - 正規表現駆動型 論理回路シミュレータ

Pythonの正規表現を用いた、独自のデジタル論理回路シミュレーションエンジン（簡易ハードウェア記述言語インタープリタ）です。

通常のPython実行ではなく、回路ファイル（`.py`）のソースコードを文字列として1行ずつ正規表現で解析し、対応する論理ゲート関数を動的にマッチングして実行します。

## 特徴
- **完全独立設計**: 入力宣言（`inname`）、論理演算（ゲート）、出力命名（`outname`）の処理が完全に分離。
- **動的パース（リフレクション）**: 正規表現で検出したゲート文字列（例: `"XOR"`）から、実際のPython関数オブジェクトを動的にルックアップして実行。
- **16bit計算機対応**: 抽象的な高レベル関数に頼らず、基本論理ゲートの組み合わせだけで「16ビット・リップルキャリー加算器（CPUのALU基盤）」を完全再現。

## 🔌 搭載論理ゲート
回路設計ファイル（`calculator.py`）内で使用できる純粋論理ゲートです。すべて多入力に対応しています。


| ゲート名 | 役割 | 動作 |
| :--- | :--- | :--- |
| `NOT(a)` | 否定（インバータ） | 入力を反転する（1→0 / 0→1） |
| `AND(*args)` | 論理積 | すべての入力が1のときだけ1を出力する |
| `OR(*args)` | 論理和 | いずれかの入力が1つのとき1を出力する |
| `XOR(*args)` | 排他的論理和 | 入力の1の数が奇数個のとき1を出力する（足し算の「答えの位」を決定） |
| `NAND(*args)` | 否定論理積 | ANDの計算結果を反転して出力する |
| `NOR(*args)` | 否定論理和 | ORの計算結果を反転して出力する |
| `XNOR(*args)` | 排他的否定論理和 | XORの計算結果を反転して出力する |

## 📁 ファイル構成
- `kairo.py`: シミュレータのコアエンジン（論理ゲート群と正規表現パーサー）
- `calculator.py`: 16ビット計算機の回路設計図（基本ゲートを1行ずつ数珠繋ぎに配線）
- `main.py`: `calculator.py` をエンジンに投入して実行するスクリプト
- `.gitignore`: 不要なキャッシュファイルをGit管理から除外する設定

## 🚀 使い方
リポジトリをクローンまたは作成し、メインスクリプトを実行します。

```bash
python main.py
```

### 🧮 動作検証（16ビット加算テスト）
初期状態の `calculator.py` には、以下の10進数の足し算が2進数ビットに変換されてあらかじめセットされています。

- **入力A**: `12345` （2進数: `0011 0000 0011 1001`）
- **入力B**: `6789`  （2進数: `0001 1010 1000 0101`）
- **期待される合計値**: `19134` （2進数: `0100 1010 1011 1110`）

正常に実行されると、不要なログを一切挟まず、16ビットの計算結果（最下位ビット `Ans0` 〜 最上位ビット `Ans15`）と溢れフラグ（`Overflow`）が、重複なしで綺麗に1回ずつ出力されます。
